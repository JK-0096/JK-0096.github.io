Project Name: Structure Starter

Project Summary

A website for a person who wants to learn beginner JavaScript concepts, which includes these five lessons:

  • Data Types
  • Variables/Constants
  • Basic Mathematical Operations
  • Conditionals
  • Loops

Intended Users

  • Absolute beginners
  • Students taking introductory courses
  • Self-learners who want quick examples

Primary Content (Client Requests)

  • Data Types: "I don't want code, just explanations."
  • Variables/Constants: "Show how to create a variable and a constant with let and const, and say the difference between them."
  • Basic Mathematical Operations: "Show sum, subtraction, multiplication and division."
  • Conditionals: "Talk about if and else and else if for several conditions and maybe logical operators."
  • Loops: "A basic example of a for loop or something."
  • Dynamic Functionality: "Also I want a button which makes the entire screen black and a creepypasta png appears on the screen."

Planned Webpages (Simplified)

  • Home (index.html): Intro, tabs to lessons, and the Easter-egg button that turns the screen black and displays giga.png.
  • Data Types (data_types.html): Explanations only (no code required per client request).
  • Variables & Constants (variables.html): Show let and const examples and explain the difference.
  • Math Operations (math_ops.html): Examples of addition, subtraction, multiplication, division.
  • Conditionals (conditionals.html): if, else if, else, and logical operators with an interactive tester.
  • Loops (loops.html): Basic for loop example and a button that prints loop output.

Page Design Details (Concise)

  • Audience: Beginner
  • Inputs: Minimal client-side inputs for demos. No server storage.
  • Validation: Client-side checks.
  • Controls: Buttons to run examples, show/hide outputs, and the Easter-egg button on the home page.
  • Navigation: Single-click nav bar to each page; consistent header/footer via components.

Dynamic Functionality (JavaScript)

Planned client-side interactivity implemented using vanilla JS and small helper libraries:

  • Easter Egg (Home): Button toggles a full-screen black overlay and displays /ITIS3135/images/creepypasta.png.
  • Conditional Tester (Conditionals page): User inputs a number or text and a script displays which branch executes (if, else if, else).
  • Loop Demo (Loops page): Button runs a for loop and appends the results to the DOM.
  • Interactive Examples Container: Use a jQuery UI accordion (or similar) to show/hide example blocks for readability.
  • AJAX Usage: Small, read-only AJAX calls to load example snippets or JSON examples from static files (e.g., /ITIS3135/data/examples.json).

Examples/inspiration: W3Schools examples, MDN learning pages

Notes & Constraints

  • No server-side CRUD — all interactivity is read-only or client-side per course rules.
  • Pages must include consistent header/footer components and validation links in the footer.
  • All forms and interactive inputs will use client-side validation only.
